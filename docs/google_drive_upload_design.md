# Google Drive 자동 업로드 기능 설계 문서

## 1. 개요
전사 완료 후 생성된 4종 파일(MP3, TXT, JSON, SRT)을 사용자의 Google Drive에 자동 업로드하는 기능을 구현한다. 로컬 전사 처리 이후, 기존에 정립된 Google Drive 폴더 구조에 맞추어 자동으로 분류하고 저장하는 것을 목표로 한다.

## 2. 기존 Google Drive 구조
사용자가 관리하는 기존 Drive 구조를 최우선으로 사용하여 업로드 경로를 매핑한다.

```text
내 드라이브/
├─ 2026 제37회 공인중개사 자격시험/
│  ├─ 기출문제/
│  ├─ 복습자료/
│  └─ 전사자료/
│     ├─ 개념완성/
│     │  ├─ [1차] 민법/
│     │  ├─ [1차] 부동산학개론/
│     │  ├─ [2차] 공인중개사법/
│     │  ├─ [2차] 부동산공법/
│     │  ├─ [2차] 부동산공시법/
│     │  └─ [2차] 부동산세법/
│     ├─ 기본이론/
│     └─ 기초이론/
├─ 기타/
├─ Colab Notebooks/
├─ NotebookLM_운영규칙_생성규칙_호출문_분리본/
└─ 영양제_관리_대장
```

## 3. 업로드 경로 설계
새로운 루트 폴더를 생성하지 않고, 기존 `내 드라이브/2026 제37회 공인중개사 자격시험/전사자료` 폴더를 기본 루트로 활용한다.

**업로드 경로 규칙:**
`내 드라이브/2026 제37회 공인중개사 자격시험/전사자료/[과정명]/[과목 폴더]/[강의 파일명 기준 폴더]/`

**예시:** `개념완성_민법_4주차_1강.mp3` 파일 전사 시
```text
내 드라이브/
└─ 2026 제37회 공인중개사 자격시험/
   └─ 전사자료/
      └─ 개념완성/
         └─ [1차] 민법/
            └─ 개념완성_민법_4주차_1강/
               ├─ 개념완성_민법_4주차_1강.mp3
               ├─ 개념완성_민법_4주차_1강.txt
               ├─ 개념완성_민법_4주차_1강.json
               └─ 개념완성_민법_4주차_1강.srt
```

## 4. 업로드 대상 파일
각 강의 폴더에 다음 4종의 파일을 일괄 업로드한다.
1. **원본 MP3**: 대용량 파일이므로 Google Drive API의 Resumable Upload(이어올리기) 방식을 기본으로 사용한다.
2. **전사 TXT**
3. **전사 JSON**
4. **전사 SRT**

## 5. 자동 분류 기준

### 5.1. 과정명 자동 분류
파일명 또는 로컬 폴더 경로에서 다음 키워드를 감지하여 매핑한다.
- `개념완성` → 전사자료/개념완성
- `기본이론` → 전사자료/기본이론
- `기초이론` → 전사자료/기초이론

**과정명 감지 실패 시 정책:**
- 기본값으로 `전사자료/분류대기` 폴더를 생성하여 사용한다.
- 또는 GUI 설정에서 사용자가 '기본 업로드 과정명'을 직접 지정할 수 있도록 지원한다.

### 5.2. 과목 자동 분류
파일명 또는 로컬 경로 키워드에 따라 과목 폴더를 매핑한다.
- `민법`, `김덕수` → `[1차] 민법`
- `부동산학개론`, `학개론`, `개론`, `이종호` → `[1차] 부동산학개론`
- `공인중개사법`, `중개사법`, `중개실무`, `정지웅` → `[2차] 공인중개사법`
- `부동산공법`, `공법`, `김희상` → `[2차] 부동산공법`
- `부동산공시법`, `공시법`, `등기법`, `지적법`, `박윤모` → `[2차] 부동산공시법` (⚠️ `공법` 키워드와 겹치지 않도록 먼저 평가)
- `부동산세법`, `세법`, `정석진` → `[2차] 부동산세법`

**과목 감지 실패 시 정책:**
- `과목불명` 또는 `분류대기` 폴더를 생성하여 해당 위치로 업로드한다.

### 5.3. 강의 단위 폴더 정책
각 강의 단위별로 MP3 파일명(확장자 제외)과 동일한 이름의 하위 폴더를 생성하여 해당 폴더 내에 4종 세트(MP3/TXT/JSON/SRT)를 보관한다.

## 6. Google Drive API 인증 설계
1. **방식**: OAuth Desktop App 기반 사용자 인증 (최초 1회 브라우저 로그인 필요)
2. **권장 Scope**: `https://www.googleapis.com/auth/drive.file` (앱이 직접 생성/사용한 파일 및 폴더만 접근 권한 부여). 보안상 전체 권한(`drive`)은 1차 구현에서 제외한다.
3. **토큰 관리**: 최초 로그인 시 획득한 token은 파일로 저장하여 이후 재사용하며, 만료 시 Refresh token을 이용해 자동 갱신 처리한다.

### Credentials / Token 보관 정책
보안 및 깃허브 업로드 방지를 위해 다음 원칙을 반드시 준수한다.
- `credentials.json` 및 `token.json`은 Git 커밋 및 업로드 절대 금지.
- **권장 보관 경로 (A안)**: 사용자가 GUI에서 `credentials.json` 파일을 선택하면 `%APPDATA%\전사도우미\` 내부에 안전하게 복사하여 사용. (예: `%APPDATA%\전사도우미\google_drive_token.json`)

## 7. 중복 파일 처리 정책
동일한 강의 단위 폴더 내에 동일한 이름의 파일이 이미 존재하는 경우의 처리 방법:

**추천 정책 (Update or Create):**
1. 파일 ID를 검색하여 동일 파일이 발견되면 **새 버전으로 업데이트 (Update)**.
2. 발견되지 않으면 **새로 업로드 (Create)**.
3. (안전 대안) 1차 구현 시나 업데이트 실패에 대비하여 동일 파일명 발견 시 `_dup1`, `_dup2` 식의 Suffix를 붙여 **새 파일로 안전하게 생성 (Create)** 하는 대안 옵션도 유지.

## 8. 업로드 실패 처리 정책
- **전사 우선 원칙**: Google Drive 업로드 실패가 로컬 전사 저장 단계나 전체 작업의 실패로 이어지지 않아야 한다. 로컬 결과물은 무조건 안전하게 보존된다.
- **부분 실패**: MP3 업로드가 실패(네트워크 등)하더라도 TXT/JSON/SRT 업로드는 멈추지 않고 계속 진행한다.
- **재시도**: 네트워크 오류 시 N회(예: 3회) 재시도 로직을 적용한다.
- **인증 오류**: Refresh 실패 시 사용자에게 재로그인(GUI 알림/로그)을 안내한다.
- **로깅**: 파일별 업로드 성공/실패 여부를 철저히 로그로 남긴다.

## 9. 구현 단계 제안
- **1단계**: Google Drive API 업로드 설계 문서 작성 (현재)
- **2단계**: `google_drive_uploader.py` 모듈 추가 (OAuth 인증, 토큰 저장/갱신, 폴더 트리 구조 탐색/생성, MP3/TXT/JSON/SRT 파일 번들 및 Resumable 업로드 로직 구현)
- **3단계**: 단독 CLI 업로드 테스트 (테스트 폴더 4종 파일 대상, 기존 Drive 폴더 트리 생성 확인)
- **4단계**: `auto_transcribe.py` 전사 완료 파이프라인에 자동 업로드 연결 (설정 On 상태일 때만 동작, 업로드 실패 시에도 전사 로직 보호)
- **5단계**: `gui_main.py`에 Google Drive 설정 UI 추가 (자동 업로드 On/Off, MP3 포함 여부, Credentials 파일 선택 기능, 기본 업로드 루트 설정)
- **6단계**: PyInstaller 패키징 옵션 수정 (Google API Client 패키지 추가, 인증 파일 보안 제외, AppData 경로 기준 런타임 적용)
- **7단계**: exe 빌드 및 GUI 실사용 업로드 통합 테스트

## 10. 설정 파일 설계
사용자 설정은 로컬 AppData 경로에 보관한다.
- **경로**: `%APPDATA%\전사도우미\settings.json`

**설정 구조 예시:**
```json
{
  "google_drive_upload_enabled": false,
  "google_drive_include_mp3": true,
  "google_drive_root_folder_name": "2026 제37회 공인중개사 자격시험",
  "google_drive_transcript_folder_name": "전사자료",
  "google_drive_duplicate_policy": "update_or_create",
  "google_drive_credentials_path": "",
  "google_drive_token_path": ""
}
```

## 11. 보안상 Git 관리 정책 (추후 구현)
구현 단계 진행 시 `.gitignore`에 다음을 추가하여 민감 정보 유출을 방지한다.
```gitignore
credentials.json
token.json
google_drive_token.json
google_credentials.json
*.client_secret.json
```
*(참고: 본 문서 작성 단계에서는 `.gitignore`를 수정하지 않는다.)*
