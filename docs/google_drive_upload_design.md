# Google Drive 자동 업로드 기능 설계 문서

## 1. 개요
전사 완료 후 생성된 4종 파일(MP3, TXT, JSON, SRT)을 사용자의 Google Drive에 자동 업로드하는 기능을 제공한다. 로컬 전사 처리 이후, 기존에 정립된 Google Drive 폴더 구조에 맞추어 자동으로 분류하고 저장한다.
현재 업로드는 `auto_transcribe.py --upload-drive` 옵션으로 실행되며, 결과 저장 완료 후 `FILE_DONE` 직전에 수행된다. 업로드 중에는 `97% Google Drive 업로드 중` 진행률 상태가 표시된다.
업로드 중 발생하는 이벤트(`DRIVE_UPLOAD_START`, `DRIVE_UPLOAD_DONE`, `DRIVE_UPLOAD_BLOCKED`, `DRIVE_UPLOAD_FAIL`)를 통해 GUI가 연동되며, 실패나 차단 시에도 전사 결과 저장은 유지된다.

## 2. 기존 Google Drive 구조
사용자가 관리하는 실제 Drive 구조를 최우선으로 사용하여 업로드 경로를 매핑한다. 각 과목 폴더 안에는 강의별 폴더가 아닌 주차별 폴더가 존재한다.

```text
내 드라이브/
└─ 2026 제37회 공인중개사 자격시험/
   ├─ 기출문제/
   ├─ 복습자료/
   └─ 전사자료/
      ├─ 개념완성/
      │  ├─ [1차] 민법/
      │  │  ├─ 개념완성_민법_1주차/
      │  │  ├─ 개념완성_민법_2주차/
      │  │  ├─ 개념완성_민법_3주차/
      │  │  └─ 개념완성_민법_4주차/
      │  ├─ [1차] 부동산학개론/
      │  ├─ [2차] 공인중개사법/
      │  ├─ [2차] 부동산공법/
      │  ├─ [2차] 부동산공시법/
      │  └─ [2차] 부동산세법/
      ├─ 기본이론/
      └─ 기초이론/
```

## 3. 업로드 경로 설계
새로운 루트 폴더를 생성하지 않고, 기존 `내 드라이브/2026 제37회 공인중개사 자격시험/전사자료` 폴더를 기본 루트로 활용한다. 강의별 하위 폴더는 별도로 만들지 않는다.

**업로드 경로 규칙:**
`내 드라이브/2026 제37회 공인중개사 자격시험/전사자료/[과정명]/[과목 폴더]/[주차 폴더]/`

**예시:** `개념완성_민법_1주차_1강.mp3` 파일 전사 시
```text
내 드라이브/
└─ 2026 제37회 공인중개사 자격시험/
   └─ 전사자료/
      └─ 개념완성/
         └─ [1차] 민법/
            └─ 개념완성_민법_1주차/
               ├─ 개념완성_민법_1주차_1강.mp3
               ├─ 개념완성_민법_1주차_1강.txt
               ├─ 개념완성_민법_1주차_1강.json
               ├─ 개념완성_민법_1주차_1강.srt
               ├─ 개념완성_민법_1주차_2강.mp3
               ├─ 개념완성_민법_1주차_2강.txt
               ├─ 개념완성_민법_1주차_2강.json
               └─ 개념완성_민법_1주차_2강.srt
```

## 4. 업로드 대상 파일
해당 주차 폴더 안에 다음 4종의 파일을 직접 일괄 업로드한다.
1. **원본 MP3**: 대용량 파일이므로 Google Drive API의 Resumable Upload(이어올리기) 방식을 기본으로 사용한다.
2. **전사 TXT**
3. **전사 JSON**
4. **전사 SRT**

## 5. 자동 분류 기준

### 5.1. 과정명/과목명 분류 (GUI 선택 우선)
자동 분류의 최우선 기준은 **GUI에서 사용자가 선택한 과정명과 과목명**이다. 다운로드 원본 파일명에는 과목명이 없는 경우가 많아, GUI 선택값을 우선하여 표준 파일명으로 정규화한 뒤 업로드를 수행한다.

선택한 과정명/과목명은 **이번 전사 대상 파일 전체**에 적용되므로, 여러 과목 파일이 한 폴더에 섞여 있다면 전체 전사를 바로 실행하지 말고 과목별로 파일을 체크해서 나누어 실행해야 한다.
예: 민법 체크 후 `개념완성/민법` 전사 → 학개론 체크 후 `개념완성/부동산학개론` 전사

GUI 선택값이 없거나 불분명할 경우 파일명 또는 로컬 폴더 경로 키워드에서 차선으로 매핑을 시도한다.
과정/과목 감지 실패(분류대기, 과목불명)나 주차/강 번호 누락 시 Drive 업로드는 차단된다.

파일명 또는 로컬 경로에서 주차를 의미하는 패턴을 감지하여 해당 주차 폴더로 분류한다.
- 예: `개념완성_민법_1주차_1강.mp3` → 주차 폴더: `개념완성_민법_1주차`
- 예: `개념완성_민법_4주차_3강.mp3` → 주차 폴더: `개념완성_민법_4주차`
- 예: `기본이론_공법_7주차_2강.mp3` → 주차 폴더: `기본이론_공법_7주차`
- 예: `기초이론_세법_2주차_1강.mp3` → 주차 폴더: `기초이론_세법_2주차`

**감지 로직 및 실패 시 정책:**
- 파일 stem에서 `_숫자주차_숫자강` 패턴을 정규식 등으로 찾는다.
- `..._1주차_3강` 형태이면 `_3강` 앞부분까지를 잘라 주차 폴더명으로 사용한다.
- 감지 실패 시 파일 stem을 그대로 쓰지 않고 `주차불명` 폴더로 업로드하거나 GUI 설정값을 사용한다.

## Google Drive 업로드 전 파일명 정규화 정책 (구현 완료)

GUI 과정/과목 선택값을 바탕으로 전사 시작 전 대상 MP3 파일명을 표준 형식으로 정규화한다.

### 표준 업로드/전사 파일명
- Google Drive 업로드 및 전사 결과 저장의 기준 파일명이다.
- 형식은 `과정명_과목명_N주차_N강`으로 통일한다.
- 예: `개념완성_부동산세법_4주차_1강`

표준 형식 변환 규칙:
- 다운로드 원본 파일명에는 페이지, 날짜, `+` 기호 등이 포함되어 있다. 정규화 시 페이지 표기와 불필요한 기호를 모두 제거한다.
- 강 번호는 폴더 내 natural sort 기준으로 1강, 2강, 3강 순차 부여하거나, 부동산학개론처럼 명시적 `[N-M]` 패턴(`[4-1] → 4주차_1강`)이 있을 경우 이를 우선 적용한다.
- 전사 시작 전 기존 결과물(TXT, JSON, SRT)이 이미 존재한다면 MP3 정규화와 동일한 stem으로 함께 변경된다.



### Google Drive 업로드와의 관계
- Google Drive 업로드는 표준화된 파일명을 기준으로 수행한다.
- 표준화되지 않은 MP3는 업로드 전에 정규화 대상이다.
- `google_drive_uploader.py`의 과정/과목/주차 감지는 표준 파일명을 기준으로 안정적으로 동작하도록 설계한다.
- 표준 파일명 미적용 상태에서 Drive 업로드를 진행하지 않는 방향으로 설계한다.

### CURRENT PROGRESS 및 트레이 알림 진행률 동기화
- 현재 로컬 Whisper 전사 중 CURRENT PROGRESS 바가 전사 중 채워지지 않는 문제가 있다.
- 사용자가 보기에는 프로그램이 멈춘 것처럼 보일 수 있으므로 Google Drive 자동 업로드 전 현재 파일 진행률 표시를 개선한다.
- 로컬 Whisper는 정확한 실시간 진행률을 제공하기 어렵기 때문에 1차 구현에서는 단계 기반 진행률을 사용한다.
- 권장 단계:
  - 파일 시작: 5%
  - 모델 준비 완료: 15%
  - Whisper 전사 시작: 20%
  - Whisper 전사 완료: 75%
  - corrections 적용 완료: 85%
  - TXT/JSON/SRT 저장 완료: 95%
  - `FILE_DONE` 수신: 100%
- CURRENT PROGRESS를 수정할 때 메인 화면의 현재 파일 진행률 바만 수정하지 않고 작업 트레이 알림/Toast에 표시되는 진행률 바도 같은 기준으로 동기화한다.
- 동기화 대상:
  - 메인 화면 CURRENT PROGRESS 바
  - 메인 화면 현재 파일 퍼센트 텍스트
  - 작업 트레이/Toast 진행률 바
  - 작업 트레이/Toast 진행률 퍼센트 텍스트
  - 현재 파일명 표시
- `auto_transcribe.py`에서 진행 단계 이벤트를 출력하고, `gui_main.py`가 이를 받아 메인 UI와 Toast UI를 같이 갱신하는 방향으로 설계한다.
- 예시 이벤트:
  - `FILE_PROGRESS|filename|5|파일 시작`
  - `FILE_PROGRESS|filename|20|Whisper 전사 시작`
  - `FILE_PROGRESS|filename|75|Whisper 전사 완료`
  - `FILE_PROGRESS|filename|85|corrections 완료`
  - `FILE_PROGRESS|filename|95|결과 저장 완료`
  - `FILE_DONE|filename` 수신 시 100%
- 향후 Google Drive 업로드가 붙으면 업로드 단계도 현재 파일 진행률에 포함할 수 있다.
  - 로컬 저장 완료: 90%
  - Drive 업로드 시작: 92%
  - Drive 업로드 완료: 100%
- 다만 1차 진행률 개선에서는 Google Drive 업로드 전 단계이므로 로컬 전사/저장 기준까지만 반영한다.

### REMAINING TIME 예측 개선
- 현재 REMAINING TIME이 실제보다 과하게 길게 표시되는 문제가 있다.
- 예: 표시 약 25분, 실제 전사 시간 약 7분.
- 부동산학개론 실측값은 음원 길이 54분 59초, 실제 전사 소요시간 약 10분이다.
- 부동산학개론 실측 ratio는 `10분 / 54.98분 ≈ 0.18`이다.
- 현재 기본값이 실제 RTX 5060 + fp16 환경 기준으로 너무 보수적일 수 있으므로 초기 ETA 기본 ratio는 `0.18` 수준을 우선 검토한다.
- 전사 완료 후 실제 처리시간과 음원 길이를 비교해 실측 ratio를 계산한다.
- 공식: `실제 처리시간 / 음원 길이 = 실제 전사 ratio`
- 예: 음원 50분, 실제 전사 7분이면 `ratio = 7 / 50 = 0.14`
- 첫 파일은 기본 ratio로 예측하고, 이후 파일부터는 최근 완료 파일의 실측 ratio를 반영한다.
- 최근 3~5개 평균 또는 누적 평균을 사용하는 방향을 검토한다.
- CURRENT PROGRESS가 계속 0%에 머물면 elapsed 기반 ETA 보정도 부정확해지므로 CURRENT PROGRESS 개선과 REMAINING TIME 개선은 함께 고려한다.

## 6. Google Drive API 인증 정책 (구현 완료)
1. **방식**: OAuth Desktop App 기반 사용자 인증 (최초 1회 브라우저 로그인 필요)
2. **인증 파일 보관 정책**:
   - `google_credentials.json` 및 `google_drive_token.json` 파일은 `%APPDATA%\전사도우미\` 내부에 안전하게 보관하여 사용한다.
   - credentials/token 파일은 패키징(build/dist) 및 저장소 커밋 내역에 절대 포함하지 않는다.
3. **토큰 무결성 검증**:
   - GUI는 실행 시 인증 파일 존재 여부 및 JSON 구조(크기, 필수 키)를 읽기 전용으로 검사한다.
   - 파일이 없거나 손상된 경우 Google Drive 자동 업로드 옵션 적용을 차단하고 사용자에게 상황형 알림창("Google Drive 인증이 필요해요")을 띄워 안내한다.

## 7. 주요 구현 함수 설계

### 7.1. Drive 경로 생성 함수 (`build_drive_folder_path`)
- **역할:** 파일명을 기반으로 Google Drive 내의 업로드 대상 폴더 경로(리스트)를 반환한다.
- **반환 경로 예시:**
  `[ "2026 제37회 공인중개사 자격시험", "전사자료", 과정명, 과목폴더, 주차폴더 ]`
- 강의별 별도 하위 폴더는 생성하지 않는다.

### 7.2. 번들 업로드 함수 (`upload_transcription_bundle`)
- **설명:**
  - `upload_transcription_bundle(audio_path, include_mp3=True, service=None)`
  - MP3 파일명을 기준으로 같은 stem을 공유하는 `.txt`, `.json`, `.srt` 파일을 찾는다.
  - Drive에서는 과정/과목/주차 폴더까지만 생성 또는 탐색한다.
  - 해당 주차 폴더 안에 MP3/TXT/JSON/SRT 4종 파일을 직접 업로드한다.
  - 강의별 별도 폴더는 생성하지 않는다.

## 8. 중복 파일 처리 정책
동일한 주차 폴더 안에 동일한 이름의 파일이 이미 존재하는 경우의 처리 방법:

**추천 정책 (Update or Create):**
1. 같은 주차 폴더 안에 같은 파일명이 발견되면 **새 버전으로 업데이트 (Update)**.
2. 없으면 **새로 업로드 (Create)**.
3. (안전 대안) 1차 구현 시나 업데이트 실패에 대비하여 동일 파일명 발견 시 `_dup1`, `_dup2` 식의 Suffix를 붙여 **새 파일로 안전하게 생성 (Create)** 하는 대안 옵션도 유지.

## 9. 업로드 실패 처리 정책
- **전사 우선 원칙**: Google Drive 업로드 실패가 로컬 전사 저장 단계나 전체 작업의 실패로 이어지지 않아야 한다. 로컬 결과물은 무조건 안전하게 보존된다.
- **부분 실패**: MP3 업로드가 실패(네트워크 등)하더라도 TXT/JSON/SRT 업로드는 멈추지 않고 계속 진행한다.
- **재시도**: 네트워크 오류 시 N회(예: 3회) 재시도 로직을 적용한다.
- **인증 오류**: Refresh 실패 시 사용자에게 재로그인(GUI 알림/로그)을 안내한다.
- **로깅**: 파일별 업로드 성공/실패 여부를 철저히 로그로 남긴다.

## 10. 구현 이력 (최근 완료 항목)
- **GUI 파일명 정규화**: 과정/과목 선택 우선 적용, natural sort 기반 강 번호 추출, 충돌 검증 및 미리보기 도입
- **Google Drive 자동 업로드**: `--upload-drive` 인자 기반 업로드, 토글 옵션 연동, 4종 결과물 동시 업로드
- **UI/UX 개선**: 왼쪽 사이드바 영역 분리(파일 분류, Drive 업로드), QScrollArea 적용(잘림 방지)
- **알림창 톤 매너 통일**: 아이콘-제목 가로 정렬, 상황 중심 텍스트로 알림창 개편 (`show_app_message` 헬퍼 적용)
- **PyInstaller 패키징**: `google_drive_uploader.py` 모듈 및 API 클라이언트 의존성 정상 포함 확인

## 11. 설정 파일 설계
사용자 설정은 로컬 AppData 경로에 보관한다.
- **경로**: `%APPDATA%\전사도우미\settings.json`

**설정 구조 예시:**
```json
{
  "google_drive_upload_enabled": false,
  "google_drive_include_mp3": true,
  "google_drive_root_folder_name": "2026 제37회 공인중개사 자격시험",
  "google_drive_transcript_folder_name": "전사자료",
  "google_drive_duplicate_policy": "update_or_create",
  "google_drive_credentials_path": "",
  "google_drive_token_path": ""
}
```

## 12. 보안상 Git 관리 정책 (추후 구현)
구현 단계 진행 시 `.gitignore`에 다음을 추가하여 민감 정보 유출을 방지한다.
```gitignore
credentials.json
token.json
google_drive_token.json
google_credentials.json
*.client_secret.json
```
*(참고: 본 문서 작성 단계에서는 `.gitignore`를 수정하지 않는다.)*
